# /harness-view 대시보드 재설계 — 설계 스펙

**날짜:** 2026-06-29
**상태:** 브레인스토밍 승인 — 구현 계획 준비됨
**스킬:** `plugins/harness-floor/skills/harness-view`
**대체:** v0.7.18 첫 컷 `/harness-view` 렌더러 (markdown→HTML 엔진은 유지, 평면 탭-아코디언 레이아웃을 교체).

> 영어 정본은 `2026-06-29-harness-view-dashboard-redesign-design.md`. 두 문서가 어긋나면 영어판이 정본이다.

## 1. 문제 & 의도

`/harness-view`는 하네스 자신의 markdown/JSON 아티팩트(라이브 `/agent-all` 런 상태, 태스크
레져, 설계 스펙)를 self-contained HTML 한 파일로 컴파일한다. v0.7.18 첫 컷은 이미 markdown
본문을 실제 HTML로 렌더한다(헤딩·표·코드·링크·인용) — 이 부분은 동작한다. 그러나 원래 의도는
사용자가 (a) 스펙/태스크를 한눈에 훑고 (b) *특정* 스펙/태스크를 raw markdown이 아니라 읽기
쉬운 HTML로 읽는 **대시보드**였다. 두 갭이 이 의도를 막는다:

1. **규모에서 탐색성이 무너진다.** Specs 탭이 날짜-접두 *파일명*의 평면 아코디언이다(이 repo
   기준 50개). 제목 추출·검색·필터·그룹핑·상태 없음 — 특정 스펙을 찾으려면 똑같이 생긴 회색 바
   50개를 스크롤해야 한다. 대시보드가 아니라 파일 목록이다.
2. **마크다운 충실도가 raw 문법을 누수한다.** 태스크 체크박스 `- [x]`/`- [ ]`가 리터럴
   `[x]`/`[]` 텍스트로 렌더되고, 중첩 리스트가 한 단계로 평면화되며, 코드펜스 언어 힌트가
   버려지고, 헤딩에 앵커가 없어 긴 스펙(최대 ~29k자)을 내비게이션할 수 없다.

### 목표
- 50+ 문서를 즉시 탐색: 제목 추출, 라이브 검색, 토픽 그룹핑, 상태.
- 개별 스펙/태스크를 목차가 있는 집중 패널에서 읽기.
- 마크다운 fidelity 결함 제거(체크박스·중첩리스트·코드 언어·앵커).

### 비목표 (YAGNI)
- 데이터 소스 추가 없음 — 현행 3종(run / tasks / specs) 유지.
- 외부 라이브러리·빌드 스텝·네트워크 없음 — 의존성 0·단일 파일·어디서나 열림 속성 보존.
- 편집 없음 — 읽기 전용 뷰.
- 전문(full-text) 관련도 랭킹 없음 — 이 규모엔 부분일치 필터로 충분.

## 2. 제약 (브레인스토밍에서 확정)

| 결정 | 선택 |
|---|---|
| 범위 | 전면 대시보드 재설계: 탐색성 + 읽기 경험 + fidelity 수정 |
| 출력 형태 | 의존성 0, 단일 self-contained `index.html`, 바닐라 JS, 직접 작성 markdown 엔진(번들 라이브러리 없음) |
| 레이아웃 | **A + 오버뷰 홈 하이브리드**: 마스터-디테일(사이드바 + 읽기 패널 + TOC) + 미선택 시 기본 랜딩으로 B의 통계 카드 홈 |

## 3. 아키텍처 — 단일 파일 보존

- **출력 불변:** 여전히 `.agent-skill/html/index.html`. `bin/render.mjs`와
  `writeDashboard({ cwd })` 진입점·시그니처 **불변** → 두 소비자가 손대지 않아도 계속 동작:
  - 온디맨드 `/harness-view`
  - `/agent-all` 각 phase 체크포인트에서 best-effort 재생성(라이브 추적)
- **렌더 전략 — 서버 프리렌더 + JS 토글 (옵션 a):** Node가 모든 문서를 HTML로 미리 렌더해
  각각 숨김 `<section data-doc-id="…">`으로 임베드. 바닐라 JS가 선택된 섹션만 보이고 나머지를
  숨긴다. 결과:
  - markdown 엔진은 **서버 사이드(Node)에만** 실린다. 브라우저엔 파서 불필요 → 무의존 유지.
  - 문서 본문 텍스트가 DOM에 있으니 클라이언트 검색이 `textContent`로 처리 — 별도 검색 인덱스
    구축·임베드 불필요.
  - 기각한 대안 (b) "raw md 임베드 + JS 파서 탑재": 엔진을 클라이언트로 중복시키고 이식성
    이득 없이 복잡도만 늘린다.
- **변경 범위:** `lib/harness-html.mjs` 내부 + 내장 CSS/JS만. 새 사이드카 파일 없음, 새 의존성 없음.
- **파일 크기:** ~50 문서 인라인 ≈ 수백 KB HTML. 로컬 파일 직접 열기엔 무리 없음 — 단일 파일
  제약의 의도된 트레이드오프로 문서화.

## 4. 레이아웃 & 컴포넌트

슬림 헤더 아래 3컬럼 CSS 그리드:

```
┌───────────────────────────────────────────────────────────┐
│ header: 프로젝트 · "harness view" · run 배지 · generated  │
├───────────────┬───────────────────────────────┬───────────┤
│ 사이드바      │ 읽기 패널                     │ TOC       │
│  검색         │  (미선택 시 오버뷰 홈;        │ "on this  │
│  Run          │   선택 시 해당 문서:          │  page"    │
│  Tasks (n)    │   제목 + 메타 + 본문)         │  h2/h3    │
│  Specs (n)    │                               │  앵커     │
│   토픽 그룹   │                               │           │
└───────────────┴───────────────────────────────┴───────────┘
```

- **사이드바(네비게이션):** 상단 검색창, 이어서 **Run / Tasks / Specs** 순.
  - Run: 행 하나 → 오버뷰 홈(또는 전용 run 뷰) 선택.
  - Tasks: 태스크당 1행 — 추출 제목 + `id` + 상태 배지.
  - Specs: 토픽 family로 그룹핑(그룹별 카운트); 각 행은 **추출 제목**(파일명 아님), 날짜,
    언어 배지(EN/KO). 파일명은 hover(`title=`)로 제공하되 주 라벨 아님.
  - 활성 행 하이라이트; 선택이 읽기 패널을 구동.
- **오버뷰 홈(기본 랜딩, 미선택 시):** B 스타일 통계 카드 —
  - Run: 상태 배지 + 태스크 + phase 타임라인(Preflight→…→Loop, done/current/todo) + decisions.
  - Tasks: 상태별 롤업 카운트(done / running / todo).
  - Specs: 총 개수 + 토픽 수 + 최근 날짜.
- **읽기 패널:** 선택 문서의 제목, 메타줄(파일명 · 날짜 · 상태), 개선된 markdown 본문.
- **TOC("on this page"):** 선택 문서의 h2/h3에서 구성; 앵커 링크로 패널 내 점프. 스크롤스파이
  (현재 섹션 하이라이트)는 nice-to-have, 필수 아님.
- **반응형:** 브레이크포인트 아래에서 사이드바는 토글 뒤로 접힘; TOC는 숨길 수 있음. 좁은
  화면에선 읽기 패널이 항상 우선 surface.

## 5. 탐색 동작

- **제목 추출:** 본문 첫 `# ` 헤딩 → 제목; 없으면 파일명 slug 폴백.
- **토픽 그룹핑:** 파일명 slug를 순서 있는 known-prefix 목록과 매칭해 `family` 도출
  (`agent-all`, `visual-qa`, `harness-builder`, `harness-debug`, `harness-explore`,
  `harness-thrift`, `cross-platform`, `hook`, `decision-surfacing`, `auto-detect`,
  `native-ask`, `cli-runtime`), 없으면 `misc`. 검증: 50 specs → 12 그룹. 그룹 내 날짜 정렬.
  (prefix 목록은 모듈 한 곳에 두어 확장 쉽게.)
- **라이브 검색:** 제목 + family + 본문 텍스트 부분일치; 사이드바 목록 필터(빈 그룹 접힘).
  검색창 비우면 전체 트리 복원.
- **딥링크:** 선택 문서를 URL 해시(`#doc=<id>`)에 반영, 헤딩 앵커 주소화. → 북마크/공유 가능,
  하네스가 특정 문서로 링크 가능. 로드 시 해시가 문서를 선택; 해시 없으면 오버뷰 홈 표시.

## 6. 마크다운 fidelity 수정 (엔진)

기존 `mdToHtml`/`inlineMd` 함수에 적용, 현재 escape·`safeHref` 보안 자세 보존:

| 수정 | 이전 | 이후 |
|---|---|---|
| 태스크 체크박스 | `- [x] foo` → 리터럴 `[x] foo` | `<li><label class="task"><input type="checkbox" disabled checked> foo</label></li>` |
| 중첩 리스트 | 모든 항목 1단계 평면화 | 들여쓰기 stack으로 깊이 추적 → 올바른 중첩 `<ul>`/`<ol>` |
| 코드펜스 언어 | ` ```python ` 버림 | 언어 캡처 → 코드블록에 대문자 라벨 배지 |
| 헤딩 앵커 | 없음 | 각 헤딩에 slug `id` 부여(dedup) — TOC + 딥링크용 |
| 소프트 줄바꿈 | 단일 개행 → `<br>` | 단일 개행 → 공백(CommonMark 문단 동작) |

TOC 데이터는 본문 렌더의 부수 출력(`{ html, toc }`)으로 생성 → 헤딩 slug를 한 번만 만들어
앵커와 TOC에 재사용.

## 10. 리스크 & 완화

- **단일 파일 크기**(50+ 문서 인라인) → 수용; 완화책은 순수 서버 프리렌더(클라이언트 파서
  없음)로 파일을 정적·빠르게 유지.
- **토픽 휴리스틱 드리프트**(새 family 등장) → prefix 목록 중앙화, `misc`가 안전 catch-all,
  family 추가는 한 줄 편집.
- **소프트랩 변경**이 일부 기존 렌더 문서를 바꿀 수 있음 → 유닛테스트 갱신 + 시각 검증으로 커버.
